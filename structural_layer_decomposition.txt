APPLICATION LAYER
- Main backend entry point: main_enhanced.py.
- Additional backend entry points: main.py and main_frappe.py.
- Core routing module for the Frappe-compatible variant: routes_frappe_complete.py.
- Main application framework: Flask.
- Session/authentication approach: custom Flask session-based authentication using session['user_id'].
- Route guards in main_enhanced.py: login_required, teacher_required, admin_required.
- Route guards in main_frappe.py: login_required, role_required.
- API/auth guards in routes_frappe_complete.py: login_required, instructor_required.
- Course management logic: main_enhanced.py route handlers browse_courses, view_course, enroll_course, create_course, edit_course, manage_course, create_lesson, edit_lesson, create_chapter, create_batch, manage_batch, add_live_class.
- Quiz engine logic: main_enhanced.py route handlers view_quiz, take_quiz, quiz_question, save_quiz_answer, submit_quiz, quiz_results, create_quiz, manage_quiz, add_quiz_question.
- Certificate logic: main_enhanced.py route handlers student_certificates, view_certificate, issue_certificates.
- Assignment logic: main_enhanced.py route handlers view_assignment, submit_assignment, view_assignment_submissions, grade_submission.
- Discussion/review logic: main_enhanced.py route handlers course_discussions, create_discussion, discussion_detail, reply_discussion, course_reviews, add_course_review.
- Administrative logic: main_enhanced.py route handlers admin_dashboard, manage_users, edit_user, delete_user, manage_courses, delete_course.
- Legacy mail-enabled controller: main.py uses Flask-Mail.

DATA ACCESS LAYER
- ORM: Flask-SQLAlchemy.
- Primary schema file: models.py.
- Alternate schema file: models_frappe_complete.py for the Frappe-compatible implementation.
- Model classes in models.py: User, Course, Enrollment, Lesson, Lesson_Resource, Lesson_Progress, Quiz, Quiz_Question, Quiz_Option, Quiz_Attempt, Quiz_Answer, Assignment, Assignment_Submission, Announcement, Chapter, Batch, BatchEnrollment, Certificate, Discussion, DiscussionReply, CourseReview, LiveClass, Enrolled.

STORAGE & PERSISTENCE LAYER
- Primary database system: SQLite.
- Main database filename: elearning.db in main_enhanced.py.
- Alternate database filenames: learn.db in main.py and frappe_lms.db in main_frappe.py.
- Migration reference path: instance/elearning.db in migrate_data.py.
- Upload root directory: uploads.
- Profile picture storage path: uploads/profiles.
- Assignment submission file storage: uploads with filenames like assignment_<assignment_id>_<user_id>_<original_filename>.
- Lesson video storage: uploads with filenames like lesson_<course_id>_<original_filename>.
- Upload serving route: /uploads/<path:filename> in main_enhanced.py.
- Frappe-compatible upload root: uploads in main_frappe.py.
